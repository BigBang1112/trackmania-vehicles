# Trackmania Vehicles

This is a generated reference of every documented Trackmania vehicle tuning revision.

It is generated by a small .NET tool.

**Note that the value mappings are not perfect yet**, contribute to the [GBX.NET](https://github.com/BigBang1112/gbx-net/blob/master/Src/GBX.NET/Engines/Plug/CPlugVehicleCarPhyTuning.chunkl) repository if you find any possible mappings.

For incorrect graphs, please report them in Issues here.

## Project structure

| Path | Description |
| --- | --- |
| [Program.cs](Program.cs) | The pipeline to generate `Reference` from `Tunings` files. |
| [Tunings](Tunings) | Source `VehicleTunings.Gbx` files, one subfolder per game (`TMF`, `TMN`, `TMSX`, `TM10`, `TM2020`, `MP4`). |
| [Reference](Reference) | Generated output. Regenerated on every run. |

Inside `Reference/<Game>/<Vehicle>.VehicleTunings.Gbx`, you'll find three sibling folders per vehicle:

| Folder | Contents |
| --- | --- |
| `Raw/` | One `.txt` per revision - every property and chunk field, sorted alphabetically. |
| `Diff/` | One `.diff` per revision - what changed compared to the previous revision. |
| `Tables/` | One `.md` per revision with the same data as nicely formatted tables |
| `README.md` | Mirrors the **latest** revision from `Tables/` |

> [!NOTE]
> The revision names (`26Nov2002`, `11Fevrier2005`, `20juin2006`, ...) come straight from Nadeo's internal tuning history and roughly track real development dates.

## Build

```
dotnet run --project TrackmaniaVehicles.csproj
```

Running it regenerates the whole [Reference](Reference) folder from `*.VehicleTunings.Gbx` files are placed under [Tunings](Tunings). Drop in Gbx files from other games/versions and they'll be picked up automatically.
